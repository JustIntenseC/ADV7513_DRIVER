# README: Настройка ADV7513 через STM32F030 по I2C для передачи RGB888 → HDMI

---

## **Описание**
Данный документ описывает процесс настройки микросхемы **ADV7513** (HDMI-энкодер) через микроконтроллер **STM32F030** по интерфейсу **I2C**. Основная задача — передача видеосигнала в формате **RGB888** в HDMI-формат.

---

## **Аппаратная часть**

### **Подключение**
| ADV7513 | STM32F030 |
|---------|-----------|
| SDA     | PB7       |
| SCL     | PB6       |
| INT     | —         |
| CEC     | —         |
| RESET   | PA8       |
| PIXCLK  | PA6       |
| VSYNC   | PA7       |
| HSYNC   | PB0       |
| D0-D7   | PC0-PC7   |
| D8-D15  | —         |
| D16-D23 | —         |

- **Питание**: 3.3V для ADV7513 и STM32F030.
- **Заземление**: Общее.
- **Сброс**: Линия RESET подключена к выводу STM32F030 (PA8).

---

## **Конфигурация тактовой частоты**
- Настроить входную тактовую частоту (PIXCLK) на 25 МГц (например).
- Убедиться, что тактовая частота соответствует требованиям ADV7513 для работы в режиме RGB888 → HDMI.

---

## **Настройки ADV7513**

### **Основные регистры**
| Регистр | Значение | Описание |
|---------|----------|----------|
| 0x98    | 0x03     | Настройка входа RGB888 |
| 0x9A    | 0x70     | Настройка выхода HDMI |
| 0x9C    | 0x30     | Настройка цветовой гаммы |

---

## **FAQ**
- **Не работает передача HDMI?**
  Проверьте подключение I2C, сброс ADV7513 и правильность записи регистров.
- **Нет сигнала на экране?**
  Убедитесь, что входная частота PIXCLK соответствует настройкам ADV7513.
- **Цвета искажены?**
  Проверьте настройки регистра цветовой гаммы (0x9C).

---
## **Требования**
- STM32F030 с поддержкой I2C.
- ADV7513 с подключенными линиями I2C, PIXCLK, VSYNC, HSYNC.
- Библиотеки STM32 HAL или Standard Peripheral Library.
